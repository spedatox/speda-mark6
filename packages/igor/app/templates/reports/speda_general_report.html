Why it happened

The outage was never a code failure

It presented as a resolver bug — every lookup returned an empty result with no error reaching the client. The actual cause was disabled billing on the cloud project, which the provider reports as an ordinary empty response rather than as an authorisation failure. Eleven hours passed before anyone noticed, because nothing in the stack distinguishes “no results” from “no entitlement”.

The code fix is small: treat a zero-result response carrying the provider's quota header as an error and alert on it. The lesson is larger. A silent degradation that still produces plausible output is the failure class most likely to run unnoticed here, and it is the one worth building detection against.

Subscription growth is structural

Sentinel flagged the same category two months running. One renewal is timing; two consecutive breaches with a 22 % price rise behind one of them is a trend. Left alone it adds roughly ₺9 600 a year.
