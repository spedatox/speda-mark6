01

Problem

Every inbound event is written synchronously to the primary datastore inside the request handler. At current volume that costs a median 180 ms of request latency and binds producer availability to database availability. Two incidents last quarter came from write amplification during backfills — in both, healthy producers failed because storage was busy.

This document proposes an asynchronous path with an explicit durable buffer, and fixes the contracts, failure semantics, and rollout sequence before any code is written.

02

Goals and non-goals

Goals

  • Producer latency independent of storage availability.
  • p99 accepted-write under 25 ms at five times current peak.
  • At-least-once delivery onto idempotent consumers.
  • Replay of any 30-day window without touching producers.

Non-goals

  • Exactly-once semantics end to end.
  • Cross-region active/active replication.
  • Any change to the public producer API.
  • Migration of historical data.

05

Contracts

The envelope versions independently of the payload. Consumers ignore unknown envelope fields and route unknown schema_version majors to the dead-letter queue rather than parsing them optimistically.

{
  "event_id":       "uuid-v7",        // idempotency key, producer-generated
  "schema_version": "2.1",            // major bump = breaking
  "occurred_at":    "2026-07-21T09:14:02.881Z",
  "partition_key":  "user:1042",      // ordering scope
  "producer":       "heartbreaker/1.8.0",
  "payload":        { }               // opaque to the pipeline
}

Flows

6.1Accepted write

producer  → ingest    POST /v2/events                  auth + schema
ingest    → log       append(partition_key)            fsync ack
ingest    → producer  202 Accepted {event_id}          ≤ 25 ms p99
consumer  → store     UPSERT … ON CONFLICT DO NOTHING  dedup by event_id

6.2Consumer failure

raise             → offset uncommitted → redelivery, backoff ×5
attempts spent    → publish to DLQ with envelope + trace
DLQ depth > 0     → page on-call; replay is manual, after the fix
